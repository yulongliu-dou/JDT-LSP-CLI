## Task 4: 审查 E2E 测试 - allCommands.test.ts

**源码**: CLI 命令 (`src/cli/commands/` + `src/cli/commandHandlers.ts`)

### 发现的关键问题

**BUG 1 - hover 无位置参数会失败**
```
测试: hover MYBATIS_PROJECT.files.configuration --json-compact
```
- `hover` 命令通过 `getPosition()` 解析位置
- 没有提供 `--method`/`--symbol`，也没有 `line`/`col`
- `isSymbolMode()` 返回 false → `!line || !col` 为 true → 返回错误
- **测试期望 `output.success === true`，但实际会返回 false**

**BUG 2 - implementations 无位置参数会失败**
```
测试: impl MYBATIS_PROJECT.files.sqlSessionInterface --json-compact
```
- 同理，`implementations` 命令也需要位置或符号参数
- 测试期望 `output.success === true`，但实际会返回 false

**数据结构问题**
- `output.data.length` 的验证：`find` 命令返回 `{ symbols: [...], count: N }`，`data` 是对象而非数组
- 测试中 `output.data.length` 对于 `find` 命令应为 `output.data.symbols.length`
- 但如果 `--json-compact` 模式下 `compactData` 对 `workspaceSymbols` 命令只保留 `name, kind, location.uri, location.range.start.line`，`data` 结构是 `{ symbols: [...], count: N }` → `data.length` 是 `undefined`

### 执行动作
1. 为 hover 测试添加 `--method` 参数或文件首行位置
2. 为 implementations 测试添加 `--method` 参数或位置
3. 修复 `output.data.length` → `output.data.symbols?.length` 或 `output.data.count`
